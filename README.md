# 📘 KaniniPro -- Article Source Code Repository

## 📌 Overview

This repository serves as the **official source code collection** for
technical articles published on the **KaniniPro blog**.

The primary goal of this repo is to:

-   Maintain structured, version-controlled source code
-   Support explanations used in blog articles
-   Provide reproducible technical examples
-   Enable readers to explore, run, and experiment with the code

This repository acts as a **companion to the KaniniPro blog**, where I
write about:

-   **Data Engineering (primary focus)**
-   **Computer Science fundamentals**
-   Cloud technologies
-   Distributed systems
-   Infrastructure as Code
-   Performance optimization
-   Real-world engineering implementations

------------------------------------------------------------------------

## 🎯 Purpose of This Repository

This repo is designed to:

1.  Keep track of code examples used in blog posts\
2.  Maintain clean and organized project structures\
3.  Allow readers to clone and reproduce experiments\
4.  Track improvements and refinements over time\
5.  Serve as a long-term technical knowledge archive

Each folder corresponds to either:

-   A specific blog post\
-   A series of related articles\
-   A technical concept demonstration\
-   A proof-of-concept implementation

------------------------------------------------------------------------

## 🧱 Repository Structure

    .
    ├── article-name-1/
    │   ├── README.md
    │   ├── src/
    │   ├── terraform/
    │   ├── scripts/
    │   └── resources/
    │
    ├── article-name-2/
    │   ├── README.md
    │   └── ...
    │
    └── shared-resources/

### Folder Guidelines

-   Each article gets its own directory\
-   Every directory contains:
    -   Code
    -   Setup instructions
    -   Dependencies
    -   Execution steps
-   If infrastructure is involved, Terraform or configuration files will
    be included\
-   If datasets are required, references or sample datasets will be
    provided

## 🚀 How to Use This Repository

1.  Clone the repository:

``` bash
git clone <repo-url>
```

2.  Navigate to the article folder:

``` bash
cd article-name
```

3.  Follow the README inside that specific folder for:
    -   Setup instructions
    -   Dependencies
    -   Execution steps
    -   Expected output
